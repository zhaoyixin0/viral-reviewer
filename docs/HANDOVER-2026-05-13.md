### 6.1 企业 SWG (Zscaler / SealSuite 等) 网络环境配置

**症状**：`npm install` 在 ffmpeg-static / youtube-dl-exec 的 postinstall 上挂，报 `UNABLE_TO_GET_ISSUER_CERT_LOCALLY` 或 `ECONNRESET` 或 `UND_ERR_CONNECT_TIMEOUT`。

**根因**：企业 Secure Web Gateway 做 TLS MITM，server cert 由企业自己的中间 CA 签。Node.js 不读 Windows 证书库，需要显式指定根 CA bundle。常见误区：以为公司用 Zscaler 但实际是 SealSuite（或反之）— **不要假设**，要看真实 cert chain。

**诊断（看 server 真实证书链）**：

```powershell
$cb = { param($s,$c,$chain,$err) $chain.ChainElements | ForEach-Object { Write-Host "Subject: $($_.Certificate.Subject) | Issuer: $($_.Certificate.Issuer)" }; return $true }
$client = New-Object System.Net.Sockets.TcpClient('release-assets.githubusercontent.com', 443)
$stream = New-Object System.Net.Security.SslStream($client.GetStream(), $false, $cb)
$stream.AuthenticateAsClient('release-assets.githubusercontent.com')
$stream.Close(); $client.Close()
```

记下根 CA 的 `Subject` (如 `CN=SealSuite SWG Root CA - v1`) 和 thumbprint。

**修复（一次性 setup）**：

1. 从 Windows 证书库导出企业根 CA 到 PEM bundle：
   ```powershell
   $bundle = 'C:\Users\<you>\corporate-ca-bundle.crt'
   '' | Set-Content $bundle -Encoding ASCII
   # 替换为你诊断步骤拿到的 thumbprint
   $thumbprints = @('EDCCF9D59E751A081F51DA328BDDB8FEC55A70F3','EF835DFEEDBF7A036511CF7F878C152607D27C87')
   foreach ($tp in $thumbprints) {
     $c = Get-ChildItem Cert:\LocalMachine\Root\$tp
     Add-Content $bundle "-----BEGIN CERTIFICATE-----"
     Add-Content $bundle ([Convert]::ToBase64String($c.Export('Cert'), 'InsertLineBreaks'))
     Add-Content $bundle "-----END CERTIFICATE-----`n"
   }
   ```

2. 设永久 User 环境变量（**路径不要含空格**，cmd 子进程可能截断）：
   ```powershell
   [Environment]::SetEnvironmentVariable('NODE_EXTRA_CA_CERTS', 'C:\Users\<you>\corporate-ca-bundle.crt', 'User')
   ```

3. 重启 shell / Claude Code 让变更生效，再 `npm install`。

**验证**：
```bash
node -e "require('https').get('https://release-assets.githubusercontent.com', r=>console.log(r.statusCode))"
# 应该输出 404 或 200，**不能**报 UNABLE_TO_GET_ISSUER_CERT_LOCALLY
```

> 一段排查史：2026-05-13 接力时撞过此问题。最初假设是 Zscaler（看到 `NODE_EXTRA_CA_CERTS` 指向 Zscaler cert），但实际 cert chain 抓出来发现是 SealSuite SWG。**不要被 env var 名字误导，要看真实 cert chain。**
