### §2.2 5 ops Vercel Blob → GCS 映射表

| Op | 当前 `@vercel/blob` | GCS `@google-cloud/storage` v7 | 行为兼容细节 |
|---|---|---|---|
| `head` | `vercelHead(key) → {url, pathname, contentType, size, uploadedAt}` | `bucket.file(key).getMetadata()` → `[{name, contentType, size, updated, ...}]`；`url = publicUrl(bucketName, key)` | 404 检测：GCS SDK 抛 `{code: 404}` 或 `ApiError` with `code:404` → `isNotFound()` helper 同时检 `err.code === 404` 与 `name === "BlobNotFoundError"`（兼容旧 path） |
| `put` | `vercelPut(key, body, {access, contentType, addRandomSuffix, allowOverwrite, cacheControlMaxAge}) → {url, downloadUrl, pathname, contentType, contentDisposition}` | `addRandomSuffix` ? `finalKey = key + "-" + suffix` : `finalKey = key`；`bucket.file(finalKey).save(body, {contentType, metadata:{cacheControl}, public: opts.access==="public", resumable: false})` 返回 `{url: publicUrl(bucketName, finalKey), downloadUrl: getDownloadUrl(url), pathname: finalKey, contentType, contentDisposition: "attachment"}` | UBLA 模式下 `public` 由 bucket-level IAM 给（W3 verdict 12b3b18 D），`save({public: true})` 在 UBLA 下是 no-op 但保留传参语义。`addRandomSuffix = false` 时 GCS 默认 overwrite（与 Vercel Blob `allowOverwrite:true` 同），开 `allowOverwrite:false` 用 `file.save(..., {preconditionOpts: {ifGenerationMatch: 0}})` |
| `list` | `vercelList({prefix, limit, cursor}) → {blobs:[{url,pathname,size,uploadedAt}], cursor, hasMore}` | `bucket.getFiles({prefix, maxResults: limit, pageToken: cursor, autoPaginate: false})` → `[files, nextQuery]`；map: `blobs = files.map(f => ({url: publicUrl(bucketName, f.name), pathname: f.name, size: Number(f.metadata.size), uploadedAt: new Date(f.metadata.updated)})); cursor = nextQuery?.pageToken; hasMore = !!nextQuery` | `getFiles` 返回 `[File[], nextQuery, apiResponse]` 三元组（SDK 怪癖），必须 `autoPaginate:false` 才返回 `nextQuery` cursor |
| `del` | `vercelDel(urls: string \| string[])` | `Array.isArray(urls) ? urls.map(urlToKey) : [urlToKey(urls)]`；`Promise.all(keys.map(k => bucket.file(k).delete({ignoreNotFound: true})))` | 唯一 caller: `lib/trending/snapshot-store.ts:115` 传 `list().blobs[i].url`（绝对 URL）。**D3 决策**: `urlToKey()` 严格 prefix-match bucketName 抛 `StorageError("del_url_unmappable", ...)` |
| `getDownloadUrl` | URL pass-through + `?download=1` query 拼 | `urlToKey(urlOrKey)` → `bucket.file(key).getSignedUrl({version:"v4", action:"read", expires: Date.now() + ttlSeconds*1000, responseDisposition: opts.filename ? \`attachment; filename="\${opts.filename}"\` : "attachment"})` → 单字符串 URL | TTL 默认 `ttlSeconds = 900` (15min，frozen by W3 12b3b18 C)；SDK 用 ADC/WIF 自动签名（无 SA key） |

**新增 helper（内部，不 export）**：

```ts
// lib/storage/api.ts
function publicUrl(bucketName: string, key: string): string {
  return `https://storage.googleapis.com/${bucketName}/${encodeURI(key)}`;
}

function urlToKey(urlOrKey: string, bucketName: string): string {
  if (!urlOrKey.startsWith("http://") && !urlOrKey.startsWith("https://")) {
    return urlOrKey; // bare key passthrough
  }
  const u = new URL(urlOrKey);
  // 支持两种 GCS public URL 形态:
  // A) https://storage.googleapis.com/<bucket>/<key>
  // B) https://<bucket>.storage.googleapis.com/<key>   (less common)
  const pathPrefix = `/${bucketName}/`;
  if (u.host === "storage.googleapis.com" && u.pathname.startsWith(pathPrefix)) {
    return decodeURI(u.pathname.slice(pathPrefix.length));
  }
  if (u.host === `${bucketName}.storage.googleapis.com`) {
    return decodeURI(u.pathname.slice(1)); // strip leading /
  }
  throw new StorageError(
    "url_not_in_bucket",
    `storage url ${u.host}${u.pathname} does not belong to bucket ${bucketName}`,
  );
}
```

### §2.3 设计决策点

#### A. Storage 实例化策略

| 选项 | 描述 | 取舍 |
|---|---|---|
| **A1**（W1 倾向） | `client.ts` lazy singleton：第一次 `getStorage()` `new Storage()`，后续 reuse | ✅ Cloud Run 容器复用 HTTP keep-alive 池；✅ 与 a-1 placeholder 模式 + W3 12b3b18 A1 一致；⚠️ test reset 用 `__resetStorageForTests()` + `vi.resetModules()` |
| A2 | per-call `new Storage()` | ❌ Cloud Run 每请求重建连接池浪费；保留无意义 |
| A3 | 模块 top-level `const storage = new Storage()` | ❌ 启动期失败时 module load 即 throw，破坏 a-1 soft-fail UX（caller 容错 `if (!enabled) return null`） |

**W1 倾向：A1**。已在 W3 verdict 12b3b18 frozen。本 scope 仅 reaffirm，避免 review 中歧义。

#### B. signed URL 生成方式

| 选项 | 描述 | 取舍 |
|---|---|---|
| **B1**（W1 倾向） | SDK `file.getSignedUrl({version:"v4", action:"read", expires, responseDisposition})`，credentials 走 ADC | ✅ SDK 自动用 ADC/WIF（Cloud Run runtime SA），**无需 SA private key 本地存**；✅ V4 signature 是 GCS 当前推荐版本；✅ 与 W3 P5 verdict E (auth 模式) 一致（WIF + ADC） |
| B2 | 手写 V4 HMAC（自实现 string-to-sign + HMAC-SHA256） | ❌ 必须有 SA private key 才能 sign，与 WIF 冲突；❌ 维护成本高；除 air-gapped 环境无任何收益 |
| B3 | `IAMCredentials.signBlob` API + 手拼 URL | ✅ 支持 WIF；❌ 比 B1 多一次 IAM API 调用 (cost + latency)；❌ 与 SDK 内置等价但更复杂 |

**W1 倾向：B1**。SDK 在 WIF 环境下自动走 `IAMCredentials.signBlob` 内部实现（即 B3 但 SDK 帮你做），caller 透明。

#### C. 错误映射

| 选项 | 描述 | 取舍 |
|---|---|---|
| **C1**（W1 倾向） | 保持现 5 个 StorageError code（`head_failed` / `put_failed` / `list_failed` / `del_failed` / `download_url_requires_full_url`）；`isNotFound()` 扩为同时检 `err.code === 404` AND `err.name === "BlobNotFoundError"` AND `message.includes("No such object")` | ✅ caller 零改动（B1 baseline）；✅ 15 contract test 不动；⚠️ 内部分支稍多 |
| C2 | 引入 GCS 专用 code (`gcs_not_found` / `gcs_quota_exceeded` / `gcs_permission_denied`)，caller 更新 `catch` 分支 | ❌ 破坏 B1 zero-caller-change 契约；❌ 4 处 caller `catch` 需 audit + 测试 fixture 更新 |
| C3 | 引入新 `StorageError` 子类（`NotFoundError extends StorageError`），head 失败抛而不 return null | ❌ 破坏 W3 frozen B1（head→null）；❌ snapshot-store / topic-cache / account-profile 3 处 caller `if (!meta?.url) return null` 全 broke |

**W1 倾向：C1**。`del_url_unmappable` 是**新增 code**（D3 决策派生，不破坏现有 code 集）。

#### D. URL → key 反映射策略（`del` + `getDownloadUrl` 接受 URL）

**根因**：`lib/trending/snapshot-store.ts:115` 唯一 `del()` caller 传 `list().blobs[i].url`（绝对 URL），`getDownloadUrl()` 已 frozen 接受 `urlOrKey: string`。

| 选项 | 描述 | 取舍 |
|---|---|---|
| D1 | 宽松正则 `^https?://[^/]+/(?:[^/]+/)?(.+)$` 取最后 path 部分 | ❌ 跨 bucket 误删风险（攻击者构造 URL 让 `del()` 删错 bucket 的 key） |
| D2 | 强制 caller 传 bare key，URL 输入抛 `StorageError` | ❌ 破坏 snapshot-store contract，需 prune helper 改写；超出 b-1 zero-caller scope |
| **D3**（W1 倾向） | `new URL()` parse + bucketName 严格 prefix match（支持两种 GCS 公开 URL 形态）；mismatch 抛 `StorageError("url_not_in_bucket", ...)` | ✅ 安全（跨 bucket URL 立即 fail）；✅ caller 零改动（snapshot-store URL 就是本 bucket，永远 match）；✅ 明确错误便于 ops 排查 |

**W1 倾向：D3**。新增 1 个 StorageError code (`url_not_in_bucket`)，不破坏 C1 baseline。

#### E. addRandomSuffix 实现

**已 W3 verdict 12b3b18 frozen as I**：`crypto.randomUUID().slice(0, 8)`（8 chars hex）。本 scope 不重审。

#### F. 测试 mock 策略

| 选项 | 描述 | 取舍 |
|---|---|---|
| **F1**（W1 倾向） | `vi.mock("@google-cloud/storage")`，mock `Storage` / `Bucket` / `File` 类调用链返回 | ✅ 与 a-2 `vi.mock("@vercel/blob")` 同 pattern，迁移成本最低；✅ 离线 + 快（vitest 单 worker < 1s）；✅ 15 个 contract case 一对一 port |
