#### F. Rollback strategy

| 选项 | 描述 | 取舍 |
|---|---|---|
| **F1**（W2 倾向） | Cloud Run **自带** progressive rollout: `gcloud run services replace` 默认新 revision 100% traffic 立即切；若 post-deploy smoke test `curl /api/health` 5xx → workflow step `gcloud run services update-traffic --to-revisions=PREV=100` 自动回滚 | ✅ 显式 rollback step；✅ revision PREV 通过 `gcloud run revisions list --limit=2 --format='value(name)' | tail -1` 拿到；⚠️ 健康 check 只验 `/api/health`（不验业务路径） |

### §2.6 风险面 + 兜底 + **Ownership-dependency check** + cross-check §4

#### R1. WIF token mint 失败（GHA auth step）

- **风险**：runbook §4 步骤错（attribute-condition 拼错 / SA principalSet binding 漏）→ deploy.yml first step `google-github-actions/auth@v2` fail
- **兜底**：runbook §6.5 + Appendix A.1 troubleshooting；deploy.yml 加 fail-fast `continue-on-error: false`；GHA workflow summary 输出明确 error message

#### R2. service.yaml `${IMAGE_TAG}` / `PROJECT_ID` placeholder 替换错

- **风险**：deploy.yml yq 命令 typo → `gcloud run services replace` 报 invalid image ref
- **兜底**：替换后 `cat service.deploy.yaml | head -50` log 到 GHA step output；`gcloud run services replace --dry-run` 先验 schema

#### R3. Smoke test 5xx → auto-revert (F1 verdict 后)

- **风险**：`curl /api/health` 拿到 503 / 500 但实际 service 是新 image 启动慢（cold start > 30s startupProbe failureThreshold）
- **兜底**：smoke test 加 retry 3 次 + 5s 间隔（让 Cloud Run startup probe 完成）；retry 全失败再 auto-revert

#### R4. Multi-arch pin 失效（regression）

- **风险**：未来 buildx 升级或 runner 切换可能默认 multi-arch
- **兜底**：deploy.yml step inspect `docker manifest inspect` 验 image 只含 linux/amd64；mismatch → fail workflow

#### R5. AR repo 存储成本失控

- **风险**：每 PR + 每 main commit 都 push image，长期累积成本
- **兜底**：(独立 workflow) `artifact-registry-image-gc.yml` per P5 verdict 候选（不在本 scope，P5.8 observability scope 决定）；本 scope deploy.yml 加 step inspect repo size 警告（cost > $1/月 alert）

#### R6. PR preview revision 累积

- **风险**：A2 选 PR preview workflow 时，关 PR 不删 tag revision → 累积超 1000 revision 上限
- **兜底**：W4 P5.2.5 weekly GC cron 已覆盖（per W3 P5.2 verdict F1）；本 scope preview-deploy.yml 也可加 `pull_request: types: [closed]` trigger 即时删 PR tag revision（更 surgical）

#### R7. Secret bootstrap 时序

- **风险**：第一次 deploy 时 Secret Manager secrets 还没创建（P5.6 phase 未启） → Cloud Run service start 时 `secretmanager.secretAccessor` 找不到 secret → container fail
- **兜底**：deploy.yml 加 pre-deploy step `gcloud secrets list --filter='name:anthropic-api-key OR name:openai-api-key OR name:google-api-key OR name:apify-token OR name:blob-read-write-token' --format='value(name)' | wc -l` 验 5 个 secret 存在；缺失 → fail workflow with 明确 message 指引 user 跑 P5.6 bootstrap
