### B · `responseSchema` 强制 Gemini 输出（schema 放松的 long-term 替代）

5/12 hotfix 是放松 Zod schema 接受 `null` 和 `float`。但根因是 `lib/video/gemini-understand.ts:185-191` 调 Gemini 时没传 `responseSchema`，靠 prompt 让它按格式输出 = 不稳定。

```ts
// 现状
ai.models.generateContent({
  model: "gemini-2.5-pro",
  config: {
    responseMimeType: "application/json",
    temperature: 0.3,
  },
})
```

**Long-term 解法**：
1. 装 `zod-to-json-schema`
2. 加 `lib/cut-plan/to-gemini-schema.ts` helper（Zod → Gemini OpenAPI subset）— 注意 Gemini 不支持 `$ref` / `oneOf` / `anyOf` / `$schema`，需要 sanitize
3. 注入 `responseSchema` 到两处 generateContent：`gemini-understand.ts` 和 `analyze-potential.ts`
4. 之后可以收紧 Zod schema 回 int + non-null（因为 Gemini 强制按 schema 输出）

工作量 60-90 分钟。
